# Security Policy

## Reporting a vulnerability

Please do not open public issues for security-sensitive reports.

Report vulnerabilities privately to the project maintainers through the configured security contact path.

## Security priorities

For voice-cli, important security areas include:
- unsafe command execution
- hidden or ambiguous action approval flows
- transcript and session-history leakage
- secrets exposure through terminal capture
- privilege escalation through CLI adapters
- misleading summaries that obscure dangerous raw output

## Supported versions

During early public beta, only the latest published beta should be assumed supported.
