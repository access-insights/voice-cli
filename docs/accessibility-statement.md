# Accessibility Statement

voice-cli is being built with a high accessibility bar.

## Goals
- strong keyboard access across all primary flows
- transcript access for every spoken interaction
- clear distinction between summaries and verbatim output
- non-visual confirmation for consequential actions
- onboarding that works for non-technical users

## Non-goal
voice-cli is not a screen reader replacement. It is a conversational access layer for CLI workflows.

## Ongoing expectations
All feature work should include an accessibility impact note.
