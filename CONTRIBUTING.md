# Contributing

Thanks for contributing to voice-cli.

## Workflow

1. Open or discuss an issue first for non-trivial changes.
2. Keep PRs small and reviewable.
3. Include accessibility impact notes for UI or interaction changes.
4. Include tests or validation notes for behavioral changes.
5. Update documentation when behavior changes.

## Development priorities

- accessibility and trust before flourish
- clear summaries and explicit raw-output access
- strong keyboard and transcript access
- safe confirmation flows for risky actions

## Pull requests

Please include:
- problem statement
- approach summary
- screenshots or transcript examples if relevant
- accessibility considerations
- test or validation notes
